What is a Variable?

📖 LessonModule 1~5 min read

A variable is like a labelled box where you store information in a program. Instead of writing the same value over and over, you give it a name and reuse it.

Real-World Analogy

Imagine your phone's contact list. The name "Mom" is a variable — it stores the actual phone number. Whenever you say "call Mom," your phone looks up the number stored under that name.

Variables in Code

Here's what creating a variable looks like in JavaScript (the language of the web):

let userName = "Alex";
let age = 28;
let isMember = true;

We created three variables:

  • userName holds the text "Alex"
  • age holds the number 28
  • isMember holds true (a yes/no value)

Naming Rules

  • Use descriptive names: customerAge, not x
  • No spaces — use camelCase: firstName
  • Can't start with a number
  • Case-sensitive: Name ≠ name
💡 Pro Tip: Good variable names make code read like English. Future-you will be grateful!

Updating a Variable

let score = 0;
score = score + 10; // score is now 10
score += 5;         // score is now 15

Variables can change value — that's the whole point! The computer tracks the latest value for you.

Data Types: Text, Numbers & Booleans

📖 LessonModule 1~6 min read

Not all data is the same. A program needs to know what kind of data it's working with so it can do the right thing with it.

The Three Core Types

1. Strings (Text)

Anything wrapped in quotes is a string:

let greeting = "Hello, world!";
let city = 'New York';
let sentence = `I am ${age} years old.`; // template literal
🔤 Remember: Even numbers become text if you wrap them in quotes: "42" is text, not a number you can do math with!

2. Numbers

Numbers have no quotes and can be used in calculations:

let price = 39;
let tax = 0.08;
let total = price * (1 + tax); // 42.12

3. Booleans (True/False)

Booleans represent on/off, yes/no, true/false — exactly two possible values:

let isLoggedIn = false;
let hasPurchased = true;
let isOver18 = age >= 18; // evaluates to true or false

Why Does Type Matter?

Type determines what operations make sense:

"10" + "5" === "105"  // string concatenation!
10  + 5   === 15     // arithmetic addition

This is one of the most common beginner bugs — mixing strings and numbers.

Checking the Type

typeof "hello"  // "string"
typeof 42       // "number"
typeof true     // "boolean"
💡 Pro Tip: When in doubt, use typeof to inspect what type a variable holds.

If/Else: Making Decisions in Code

📖 LessonModule 2~7 min read

Programs need to make decisions. The if/else statement lets your code choose different paths based on conditions.

The Basic Syntax

if (condition) {
  // runs when condition is TRUE
} else {
  // runs when condition is FALSE
}

Real Example

let temperature = 85;

if (temperature > 80) {
  console.log("It's hot! Wear light clothing.");
} else {
  console.log("Comfortable weather today.");
}

Multiple Conditions: else if

let score = 75;

if (score >= 90) {
  console.log("Grade: A");
} else if (score >= 80) {
  console.log("Grade: B");
} else if (score >= 70) {
  console.log("Grade: C");
} else {
  console.log("Grade: F — keep studying!");
}

Comparison Operators

  • === equal to (strict)
  • !== not equal to
  • > greater than
  • < less than
  • >= greater than or equal
  • <= less than or equal

Logical Operators

// AND — both must be true
if (age >= 18 && hasID) { ... }

// OR — at least one must be true
if (isAdmin || isModerator) { ... }

// NOT — flips the value
if (!isLoggedIn) { ... }
💡 Always use === (strict equality) instead of == in JavaScript — it avoids type coercion surprises.

Loops: Repeating Tasks Automatically

📖 LessonModule 2~7 min read

Loops let you run the same block of code multiple times without copying and pasting. They're one of the most powerful tools in programming.

The For Loop

Use a for loop when you know how many times you want to repeat:

for (let i = 0; i < 5; i++) {
  console.log("Iteration number: " + i);
}
// Prints: 0, 1, 2, 3, 4

The three parts of a for loop:

  1. let i = 0 — start value
  2. i < 5 — keep going while this is true
  3. i++ — change the counter after each run

Looping Through Arrays

let fruits = ["apple", "banana", "cherry"];

for (let i = 0; i < fruits.length; i++) {
  console.log(fruits[i]);
}

// Modern style using for...of:
for (let fruit of fruits) {
  console.log(fruit);
}

The While Loop

Use while when you don't know in advance how many iterations you need:

let lives = 3;
while (lives > 0) {
  console.log("Lives remaining: " + lives);
  lives--;
}
// Prints: 3, 2, 1
⚠️ Infinite Loop Warning: Always make sure your loop condition will eventually become false — otherwise your program hangs!

Practical Use Case

// Sum all numbers from 1 to 100
let total = 0;
for (let i = 1; i <= 100; i++) {
  total += i;
}
console.log(total); // 5050

Functions: The Building Blocks of Programs

📖 LessonModule 3~8 min read

A function is a named, reusable block of code that performs a specific task. Functions are the single most important concept in all of programming.

Defining and Calling a Function

// Define the function
function greet(name) {
  return "Hello, " + name + "!";
}

// Call (use) the function
let message = greet("Alex");
console.log(message); // "Hello, Alex!"

Why Functions Matter

  • Reusability — Write once, use many times
  • Readability — Code reads like sentences
  • Maintainability — Fix a bug in one place
  • Abstraction — Hide complexity behind a simple name

Parameters & Return Values

function calculateTax(price, taxRate) {
  let tax = price * taxRate;
  return price + tax;
}

let total = calculateTax(39, 0.08); // 42.12

Arrow Functions (Modern Style)

// Traditional
function add(a, b) { return a + b; }

// Arrow function
const add = (a, b) => a + b;

// Usage is identical
console.log(add(3, 4)); // 7
💡 Single Responsibility Principle: Each function should do one thing and do it well. If your function needs a long name to describe it, split it up!

Functions Calling Functions

function square(n) { return n * n; }
function cube(n)   { return square(n) * n; }

console.log(cube(3)); // 27

What Is a Prompt and How AI Reads It

📖 LessonModule 4~7 min read

A prompt is the message you give to an AI model. It's the steering wheel — the quality of your prompt determines the quality of the response.

How AI Processes Your Prompt

Modern AI models like ChatGPT and Claude are trained to predict the most useful response to your input. They look for:

  • Context — What situation is this in?
  • Intent — What does the user actually want?
  • Format cues — Should the answer be a list, code, essay?
  • Constraints — Any rules or boundaries to follow?

The Difference a Prompt Makes

❌ Weak Prompt: "Write about marketing"

✅ Strong Prompt: "Write a 200-word LinkedIn post for a solopreneur launching their first online course. Tone: energetic and confident. Include a call-to-action."

Same AI, completely different output. The second prompt gives the AI:

  • A specific platform (LinkedIn)
  • An audience (solopreneur)
  • A length target (200 words)
  • Tone guidance
  • A specific element to include

The 5 Elements of a Great Prompt

  1. Role — Who should the AI be?
  2. Task — What exactly should it do?
  3. Context — What's the background?
  4. Format — How should it respond?
  5. Constraints — What to avoid or limit?
💡 Key Insight: AI doesn't read your mind. Every assumption it has to make is a chance for the output to miss the mark.

The ROLE-TASK-FORMAT Framework

📖 LessonModule 4~8 min read

The RTF Framework (Role, Task, Format) is the simplest, most powerful prompt structure you'll ever learn. Apply it and your AI outputs will improve immediately.

The Template

You are a [ROLE].
[TASK — describe what you want].
Format the response as [FORMAT].

Example 1 — Email Writing

You are a professional business email writer.
Write a polite follow-up email to a client who hasn't responded
in 5 days. Keep it brief and friendly, not pushy.
Format: 3 short paragraphs, no more than 100 words total.

Example 2 — Code Help

You are a senior JavaScript developer.
Explain how closures work to a complete beginner.
Use a simple, real-world analogy and one short code example.
Format: plain English explanation followed by a code block.

Example 3 — Content Creation

You are a viral social media copywriter who specializes in
short-form video scripts.
Create a 30-second TikTok script for a course called
"Be Your Own AI" targeting beginners aged 18-35.
Format: Line-by-line script with timestamps.

Why Roles Work

Assigning a role "primes" the AI to access the appropriate knowledge domain and communication style. A "financial advisor" persona responds very differently from a "startup founder" persona — even to the same question.

Iterating on Prompts

Great prompting is a dialogue, not a one-shot command:

  1. Send your initial RTF prompt
  2. Evaluate the response
  3. Refine: "Make it shorter" / "More formal" / "Add examples"
  4. Repeat until perfect
🔑 The Golden Rule: If you'd have to explain it to a new employee, you need to put it in the prompt.

Chain-of-Thought Prompting

📖 LessonModule 5~8 min read

Chain-of-Thought (CoT) prompting instructs the AI to reason step-by-step before giving its final answer. This dramatically improves accuracy for complex tasks.

The Magic Phrase

"Think step by step."

Adding those four words to a prompt triggers a fundamentally different mode of reasoning in the model.

Why It Works

AI models generate text one token at a time. By forcing intermediate reasoning steps into the output, the model "builds on" earlier thoughts — catching errors before committing to a final answer.

Example — Math Problem

❌ Without CoT:
Prompt: "A store has 15 items. It sells 40% and adds 8 more. How many?"
Bad answer: 17 (wrong reasoning)

✅ With CoT:
Prompt: "A store has 15 items. It sells 40% and adds 8 more. How many? Think step by step."
Step 1: 40% of 15 = 6 items sold → 15 − 6 = 9 remaining
Step 2: 9 + 8 = 17. Answer: 17 ✓

CoT for Complex Tasks

You are a business strategist.
I want to launch an online course for $39. Think step by step
through the key risks, marketing channels, and first 30-day
action plan. Be specific and actionable.

Zero-Shot vs. Few-Shot CoT

  • Zero-Shot: Just add "Think step by step." — works for most problems.
  • Few-Shot: Provide example question + reasoning + answer pairs before your actual question. More powerful for specialized domains.
💡 Power Move: Combine RTF + CoT: "You are a [ROLE]. [TASK]. Think step by step, then present your final answer in [FORMAT]."

Prompt Patterns & Templates

📖 LessonModule 5~8 min read

Master prompting is about having a toolbox of proven patterns you can reach for instantly. Here are the most powerful ones.

Pattern 1 — The Persona Pattern

Act as [persona] and [task].
Example: "Act as a skeptical venture capitalist and critique
my business idea. Be harsh but constructive."

Pattern 2 — The Flipped Interaction Pattern

I want you to ask ME questions to gather the information
you need, then [task] based on my answers.
Example: "Ask me 5 questions about my target audience,
then write a product description."

Pattern 3 — The Template Fill Pattern

Fill in this template: [template with blanks].
Example: "Fill in: The course [COURSE_NAME] helps [AUDIENCE]
achieve [OUTCOME] in [TIMEFRAME] without [OBSTACLE]."

Pattern 4 — The Context Manager

Before we start, here is the context you must always remember:
[background info]. Now, [task].

This lets you "brief" the AI once and avoid repeating yourself
in every follow-up message.

Pattern 5 — The Critic Pattern

Here is [your work]. Act as a [type of expert] and give me
your 3 harshest criticisms and 3 specific improvements.
Be direct, not polite.

Building Your Prompt Library

The best prompters keep a personal library of their most effective prompts. After this course, start saving prompts that get great results in a simple document.

🚀 The Ultimate Mega-Prompt Formula:

You are [ROLE]. Your audience is [AUDIENCE]. [CONTEXT]. [TASK]. Think step by step. Format your response as [FORMAT]. Constraints: [CONSTRAINTS]. Start with [SPECIFIC OPENER].

When you apply all these layers together, AI becomes a true extension of your thinking — not just a search engine. That's what it means to Be Your Own AI.
